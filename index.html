Progressively learning System Architecture

I'm drawn to understanding how systems work at their lowest levels — memory layouts, process lifecycles, network protocols, and the data structures that make it all efficient. Every project I build starts from first principles: custom hash maps instead of STL, fork()/exec() instead of thread pools, hand-rolled protocols instead of frameworks. This isn't just about building — it's about deeply understanding system design, low-level architecture, and the trade-offs that shape production infrastructure. Currently expanding into distributed systems, consensus algorithms, and large-scale system design patterns.
