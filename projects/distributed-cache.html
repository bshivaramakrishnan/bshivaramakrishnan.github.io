Core Features

What's Inside

Every component built from the ground up for maximum performance and educational value.

üîÑ

LRU Eviction

Custom doubly linked list combined with hashmap for O(1) get, set, and eviction. Most recently used items stay, least recently used get evicted.

O(1) all operations
‚è±Ô∏è

TTL Expiration

Min-heap priority queue tracks expiration times. Background thread efficiently removes expired entries without blocking reads.

O(log n) expiration
üíç

Consistent Hashing

Virtual nodes ensure balanced key distribution. Adding/removing nodes only remaps K/N keys on average, not the entire keyspace.

150 virtual nodes/server
üîí

Sharded Locking

16+ independent shards with per-shard read-write locks. Concurrent reads are lock-free, writes only block their shard.

Near-linear scalability
‚öñÔ∏è

Quorum Consistency

Configurable consistency levels: ONE for speed, QUORUM for balance, ALL for strong consistency. Automatic read-repair on mismatches.

ONE / QUORUM / ALL
üíì

Failure Detection

Phi accrual failure detector adapts to network conditions. No fixed timeouts ‚Äî uses statistical model of heartbeat arrivals.

Adaptive thresholds
üìä

Prometheus Metrics

Export cache hits, misses, evictions, latency percentiles, and cluster health. Ready for Grafana dashboards.

:9090/metrics
üßµ

Thread Pool

Work-stealing thread pool for async operations. Configurable worker count, task queue with priority support.

Async I/O ready
üó∫Ô∏è

Cluster Topology

Graph-based cluster representation. BFS for partition detection, Dijkstra for optimal replica routing.

Network-aware routing
