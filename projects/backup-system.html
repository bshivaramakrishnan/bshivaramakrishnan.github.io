System Design

Architecture

Nine interconnected modules forming a production-grade backup pipeline — each built from scratch with zero external frameworks.

⬡

Chunk Store

64KB fixed-size content-addressable blocks with SHA-256 hashing. Files split into chunks, stored by hash — identical data is never duplicated.

chunk_store.h · 268 LOC
⚙

Multi-Process Orchestrator

Spawns isolated worker processes via fork()/exec() with max 4 concurrent. Parent reaps children via waitpid() with WNOHANG.

orchestrator.h · 262 LOC
⬢

POSIX IPC Layer

Shared memory (shm_open/mmap), pipe-based message queues with select() timeouts, and named semaphores for worker coordination.

ipc.h · 256 LOC
◈

SQLite Metadata

WAL mode, global mutex, RAII locks, busy timeouts (5s), and exponential backoff retries. Zero SQLITE_BUSY errors under load.

database.h · 753 LOC
◇

Crypto Pipeline

AES-256-CBC encryption with per-job CSPRNG keys, random IV per chunk. SHA-256 integrity verification on every store and restore.

aes256.h + sha256.h · 276 LOC
◆

Job Scheduler

Priority queue (URGENT→LOW) with DAG dependency resolution. Cycle detection via BFS, topological ordering via Kahn's algorithm.

job_scheduler.h · 145 LOC
▣

CoW Snapshots

Hardlink-based copy-on-write snapshots for consistent backups. Automatic cleanup after job completion — no blocking on source.

snapshot.h · 174 LOC
▦

Compression Engine

Dual-mode: LZ4 for speed, Zstandard for high compression ratio. Applied per-chunk before encryption in the storage pipeline.

compressor.h · 109 LOC
▤

Restore Engine

Full file restoration: retrieve manifests → decrypt → decompress → verify SHA-256. Integrity-only mode available via --verify.

restore_engine.h · 153 LOC
