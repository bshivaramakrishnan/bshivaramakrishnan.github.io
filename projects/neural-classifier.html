CNN Architectures

Models Explored

Three architectures evaluated for grocery product classification, each with distinct trade-offs in accuracy, speed, and resource usage.

üèõÔ∏è

AlexNet

The pioneer of deep CNNs. Five convolutional layers with max pooling, followed by three fully connected layers. Uses dropout and data augmentation to combat overfitting.

~60M params ¬∑ High accuracy ¬∑ GPU recommended
‚ö°

ShuffleNet

Designed for mobile and edge devices. Uses pointwise group convolution and channel shuffle operations for computational efficiency while maintaining competitive accuracy.

~1.4M params ¬∑ Edge-friendly ¬∑ CPU viable
üîß

Custom CNN

Handcrafted 3-block architecture tailored to the grocery dataset. Uses BatchNorm, GlobalAvgPooling, and dropout for a sub-50K parameter model that trains in minutes on CPU.

~50K params ¬∑ Domain-specific ¬∑ Fastest training
üåê

MobileNet (Frontend)

Pretrained on ImageNet, used as a frozen feature extractor in the browser. A lightweight dense head is trained on your images via TensorFlow.js for real-time inference.

Transfer learning ¬∑ Browser-native ¬∑ WebGL
üîÑ

Data Augmentation

Rotation, width/height shift, shear, zoom, horizontal flip, and brightness range applied during training to improve generalization and prevent overfitting on small datasets.

Keras ImageDataGenerator ¬∑ Runtime augmentation
üéì

Transfer Learning

Fine-tune pre-trained models on the grocery dataset. Leverages representations learned from millions of images to achieve high accuracy with limited domain-specific data.

ImageNet weights ¬∑ Feature reuse ¬∑ Fast convergence

Backend ‚Äî Python

Train models offline with the full Python pipeline. Works on any machine with Python 3.8+.

terminal ‚Äî python backend

$ git clone https://github.com/bshivaramakrishnan/NEURAL_NETWORKS_IMAGE_CLASSIFIER.git
$ cd NEURAL_NETWORKS_IMAGE_CLASSIFIER

# Create virtual environment
$ python -m venv venv
$ source venv/bin/activate

# Install dependencies
